=== PD filter plugin

==== Description

The pd filter allows you to generate a non tamperable seal
to each processed event. It first generates a UUID, see 
https://en.wikipedia.org/wiki/Universally_unique_identifier[UUID]
and then concatenates the message, hashes it, and register this
UUID+Hash to the user specified blockchain. Then, we add the assetId
back to message and send it out as a processed event.

This way, an event is condered to be untampered. Ideally, this must be
the first filter that an event should go through, in order to
preserve the message's authenticity.

==== PD Filter Configuration Options

This plugin supports the following configuration options plus the <<plugins-{type}s-{plugin}-common-options>> described later.

[cols="<,<,<",options="header",]
|=======================================================================
|Setting |Input type|Required
| <<plugins-{type}s-{plugin}-user>> |<<string,string>>|Yes
| <<plugins-{type}s-{plugin}-password>> |<<string,string>>|Yes
| <<plugins-{type}s-{plugin}-endpoint>> |<<string,string>>|No
| <<plugins-{type}s-{plugin}-storage>> |<<string,string>>|No
|=======================================================================

Also see <<plugins-{type}s-{plugin}-common-options>> for a list of options supported by all
filter plugins.

We suggest you keep the storage as "public".

Example:
[source,ruby]
    filter {
      pd {
        user => "demo1", password => "Test@12345678", storage => "private"
      }
    }
